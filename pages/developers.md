## Building open-source eventing infrastructure to transform simulations into interactive experiential learning.

### Event-Driven Interoperability Extensions for the Open-Source Simulation Community

Imagine a chemical industrial process where **DWSIM** handles the continuous physics (pressures, temperatures, flow rates), while **JaamSim** handles the discrete business logic (truck arrivals, queue management, and resource scheduling).

Simulating complex phenomena—such as industrial processes, urban planning, or physical realities—rarely happens in isolation. While interoperability initiatives exist (such as OpenUSD and Functional Mockup Units), these frameworks do not yet address **event publishing**.

Practi.City aims to develop event-driven extensions for the open-source simulation community. By breaking down the silos between Continuous Process Simulators and Discrete Event Simulators, we unlock new potential for **experiential learning**.

### Are you a developer who wants to participate?

**Why join us?**
*   Do you have a passion for simulation in general?
*   Do you want to contribute to event-driven extensions for your favorite software or explore new ones?
*   Want to see your GitHub ID on the open-source community "Wall of Fame"? :-)
*   Want your bio published on our contributors page?

If so, you’re a perfect fit!

**What will your role be?**
Well... developing! But that’s not all: you can participate in our forum discussions, propose new ideas, join the testing team, write tutorials, help other developers, or curate experiential learning content.

**How can you start?**
Simply contact a core team member and explain your motivations. Include your GitHub account ID so we can add you to the team (see our contributors page).

### What is the development workflow?
1.  Be added to the Practi.City GitHub organization as a developer.
2.  We create a new repository for your project.
3.  You do the heavy lifting (the coding!).
4.  You write a comprehensive `README` so that simulation users can easily install, deploy, and use your extension.
5.  We test it end-to-end and provide feedback if needed.
6.  We merge your code into the repository's main branch.
7.  We publish a post and the "Events Manifesto" generated by your extension on the corresponding software page.
8.  We promote your extension in user group forums for wider adoption.

### What are the technical / QA requirements?
1.  We are developers too, so we will review your code quality for the final approval.
2.  Follow the **PULSE** schema (Practi.City Universal Live Simulation Events). *Note: As of April 29, 2026, we are still finalizing this standard!*
3.  Your code is open-source by definition. If it is a core extension, you must comply with the target software's existing license. If it is an external add-on, your code should be under the GNU license.
